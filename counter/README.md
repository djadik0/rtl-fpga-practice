# Задача. Счётчик с конечным автоматом

## Цель работы

Реализовать модуль `counter` на `SystemVerilog` и проверить его работу в моделировании.

## Краткое описание

В проекте реализован 3-битный счётчик, работающий на основе конечного автомата с тремя состояниями.

В зависимости от текущего состояния модуль изменяет значение выхода `out` по одному из трёх правил:

- увеличение на `1`
- ещё одно увеличение на `1`
- уменьшение на `1`

Таким образом формируется повторяющаяся последовательность действий `+1`, `+1`, `-1`.

## Что реализовано

В модуле `counter` реализованы:

- синхронная работа по положительному фронту сигнала `clk`
- асинхронный сброс по активному низкому сигналу `rst`
- внутренний автомат с тремя состояниями:
  - `sost_plus1`
  - `sost_plus2`
  - `sost_min`
- изменение значения `out` в зависимости от текущего состояния
- возврат в начальное состояние после завершения цикла состояний

### Логика работы

После сброса:

- `out` устанавливается в `0`
- автомат переходит в состояние `sost_plus1`

Далее на каждом такте выполняется цикл:

- `sost_plus1` → `out = out + 1`
- `sost_plus2` → `out = out + 1`
- `sost_min` → `out = out - 1`

После этого автомат снова возвращается в состояние `sost_plus1`.

## Что проверялось в testbench

В testbench проверялись следующие сценарии:

- генерация тактового сигнала
- начальный сброс модуля
- работа счётчика после снятия сброса
- повторный сброс во время моделирования
- повторный запуск последовательности состояний после сброса

Моделирование выполняется с простым генератором тактового сигнала и подачей сигнала сброса в начале и в середине теста.

## Результат работы

В результате был получен 3-битный счётчик с внутренним конечным автоматом, который циклически изменяет значение выхода по шаблону `+1`, `+1`, `-1`.

Работа модуля была проверена в testbench при начальном запуске и после повторного сброса.

## Вывод

В ходе выполнения задачи был реализован и проверен модуль `counter` на `SystemVerilog`.  
Моделирование показало корректную работу автомата состояний, изменение значения счётчика по заданному алгоритму и правильное восстановление начального состояния после сброса.
