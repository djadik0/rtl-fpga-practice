# Задача. 4-битный ALU

## Цель работы

Реализовать простой 4-битный модуль `ALU` на `SystemVerilog` и проверить его работу в моделировании.

## Краткое описание

В проекте реализован синхронный модуль `ALU`, который по входному коду операции `F` выполняет одну из нескольких логических или арифметических операций над входами `A` и `B`.

Результат вычисления записывается в выходной регистр `Y` по положительному фронту тактового сигнала `clk`.

## Что реализовано

В модуле `ALU` поддерживаются следующие операции:

- `000` — `A & B`
- `001` — `A | B`
- `010` — `A + B`
- `100` — `A ^ B`
- `101` — `A - B`
- `110` — `A | (~B)`
- `111` — `~A`

Для остальных значений предусмотрено значение по умолчанию:

- `default` — `0`

## Что проверялось в testbench

В testbench были поданы несколько наборов входных данных для проверки работы `ALU`.

Проверялись:

- операция `AND`
- операция `OR`
- операция сложения
- поведение для неподдерживаемого кода `011`
- операция `XOR`
- операция вычитания
- операция `A | (~B)`
- операция `~A`

Моделирование выполняется с простым генератором тактового сигнала и последовательной подачей тестовых векторов.

## Результат работы

В результате был получен простой синхронный модуль `ALU`, выполняющий базовые логические и арифметические операции над 4-битными входными данными.

Работа модуля была проверена в testbench на нескольких сценариях.

## Вывод

В ходе выполнения задачи был реализован и проверен 4-битный модуль `ALU` на `SystemVerilog`.  
Моделирование показало корректную обработку поддерживаемых кодов операций и корректное поведение для значения по умолчанию.
