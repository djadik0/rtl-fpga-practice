# Мини-проект. Автомат продажи кофе

## Цель работы

Реализовать модуль автомата продажи кофе на `SystemVerilog` и проверить его работу в моделировании.

## Краткое описание

В проекте реализован модуль `cofe_top`, моделирующий работу простого автомата продажи напитков.

Автомат принимает монеты номиналом `1`, `2`, `5` и `10`, накапливает сумму и по сигналу `select` выдаёт один из двух напитков:

- обычный кофе
- кофе с молоком

Если внесённой суммы достаточно, модуль формирует сигнал выдачи напитка и возвращает сдачу через выход `cashback`.

## Что реализовано

В модуле `cofe_top` реализованы:

- накопление внесённой суммы во внутреннем регистре `summ`
- приём монет через входы:
  - `one_coin`
  - `two_coin`
  - `five_coin`
  - `ten_coin`
- выбор типа напитка по сигналу `select`
- выдача обычного кофе по выходу `coffe`
- выдача кофе с молоком по выходу `coffe_with_milk`
- возврат сдачи через выход `cashback`
- асинхронный сброс по сигналу `rst`

### Логика работы

- при `select = 0` выбирается обычный кофе
- при `select = 1` выбирается кофе с молоком
- цена обычного кофе составляет `27`
- цена кофе с молоком составляет `33`
- после успешной покупки сумма сбрасывается в `0`
- сигналы выдачи напитка и сдачи формируются на один такт

## Что проверялось в testbench

В testbench проверялись следующие сценарии:

- начальная инициализация сигналов
- сброс автомата
- внесение монет для покупки обычного кофе
- выдача обычного кофе и проверка сдачи
- выбор кофе с молоком
- внесение монет для покупки кофе с молоком
- выдача кофе с молоком и проверка сдачи
- повторная покупка после предыдущей операции

Моделирование выполняется с простым генератором тактового сигнала и последовательной подачей сигналов монет и выбора напитка.

## Результат работы

В результате был получен модуль автомата продажи кофе с поддержкой двух режимов выбора напитка и возвратом сдачи.

Работа модуля была проверена в testbench на нескольких последовательностях внесения монет и покупки напитков.

## Вывод

В ходе выполнения мини-проекта был реализован и проверен модуль `cofe_top` на `SystemVerilog`.  
Моделирование показало корректное накопление суммы, выбор напитка, формирование сигналов выдачи и возврат сдачи после покупки.
