# Задача. Valid-Ready pipeline stage

## Цель работы

Реализовать модуль `vr_pipeline_stage` на `SystemVerilog` и проверить его работу в моделировании.

## Краткое описание

В проекте реализована одностадийная конвейерная ступень с интерфейсом `valid-ready`.

Модуль принимает входные данные `s_data_i` вместе с сигналом `s_valid_i`, сохраняет их во внутреннем регистре и передаёт дальше через выходной интерфейс `m_valid_o / m_data_o`.

Логика работы построена на стандартном механизме рукопожатия:

- входная транзакция принимается при `s_valid_i = 1` и `s_ready_o = 1`
- выходная транзакция передаётся при `m_valid_o = 1` и `m_ready_i = 1`

## Что реализовано

В модуле `vr_pipeline_stage` реализованы:

- параметризация ширины данных через `DATA_W`
- внутренняя регистрация данных во `m_data_ff`
- хранение признака валидности в `valid_reg`
- формирование входного сигнала готовности `s_ready_o`
- формирование выходного сигнала валидности `m_valid_o`
- асинхронный сброс по активному низкому сигналу `rstn_i`

### Особенности реализации

Во внутренней логике используются два сигнала рукопожатия:

- `handshake_s` — приём данных со входа
- `handshake_m` — передача данных на выход

Сигнал готовности входа формируется как:

```text
s_ready_o = !valid_reg | m_ready_i
```

Это означает, что ступень готова принять новые данные, если она пуста или если следующий блок готов принять текущие данные.

## Что проверялось в testbench

В testbench проверялись следующие сценарии:

- начальная инициализация сигналов
- сброс модуля
- передача данных при готовом приёмнике
- удержание данных при `m_ready_i = 0`
- последующая передача после повторного разрешения приёма
- подача нескольких тестовых значений на вход

Для проверки использовались значения:

- `8'h15`
- `8'h2A`
- `8'h3C`

Моделирование выполняется с простым генератором тактового сигнала и последовательной подачей входных сигналов.

## Результат работы

В результате была получена простая конвейерная ступень с интерфейсом `valid-ready`, обеспечивающая буферизацию одной транзакции.

Работа модуля была проверена в testbench в режимах свободной передачи и временной остановки по сигналу готовности приёмника.

## Вывод

В ходе выполнения задачи был реализован и проверен модуль `vr_pipeline_stage` на `SystemVerilog`.  
Моделирование показало корректную работу механизма `valid-ready`, сохранение данных во внутреннем регистре и правильное удержание транзакции до подтверждения приёма.
