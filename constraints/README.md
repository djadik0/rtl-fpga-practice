# Мини-проект. Написание SDC-констрейнтов для `MY_DESIGN`

## Цель работы

Сформировать файл временных ограничений `SDC` для заданного RTL-модуля на основе предоставленной спецификации.

В рамках задания необходимо было:
- проанализировать структуру модуля `MY_DESIGN`;
- определить тактовые, входные, выходные и комбиниционные ограничения;
- оформить их в файле `constraints_MY_DESIGN.sdc`.

## Краткое описание проекта

В задании был дан модуль `MY_DESIGN`, содержащий:
- арифметическую логику;
- внутренние регистры;
- комбиниционную логику;
- входные и выходные порты, для которых необходимо задать временные ограничения.

Основная задача состояла в том, чтобы на основе текстовой спецификации корректно записать SDC-констрейнты для анализа таймингов.

## Что было сделано

В ходе выполнения задания:
- изучена схема проекта `MY_DESIGN`;
- проанализированы входные, выходные и внутренние пути;
- сформирован файл `constraints_MY_DESIGN.sdc`;
- заданы ограничения для тактового сигнала, входов, выходов и отдельного комбиниционного пути.

## Реализованные ограничения

### Тактовые ограничения

Для сигнала `clk` были заданы:
- период тактового сигнала `3 ns`;
- `source latency` = `0.7 ns`;
- максимальная задержка распространения такта внутри дизайна = `0.3 ns`;
- `clock uncertainty` для setup = `0.29 ns`;
- максимальное время фронта тактового сигнала = `0.12 ns`.

### Ограничения на входы

Были заданы входные задержки:
- для `data1` и `data2` — `2.2 ns`;
- для `sel` — `1.4 ns`.

### Ограничения на выходы

Были заданы выходные задержки:
- для `out1` — `0.5 ns`;
- для `out2` — `0.81 ns`;
- для `out3` — `0.4 ns`.

### Ограничение на комбиниционный путь

Для пути от входов `Cin1` и `Cin2` до выхода `Cout` было задано максимальное ограничение:

```text
2.45 ns
```

## Использованные команды SDC

В работе использовались следующие основные команды:
- `create_clock`
- `set_clock_latency`
- `set_clock_uncertainty`
- `set_clock_transition`
- `set_input_delay`
- `set_output_delay`
- `set_max_delay`
